# Задача 1. Сбалансированное дерево поиска (AVL)

## Что реализовано
Реализовано **AVL‑дерево** — двоичное дерево поиска, которое поддерживает баланс по высоте:  
для любого узла разница высот левого и правого поддерева не превышает 1. Это даёт асимптотику поиска/вставки порядка **O(log n)**.

### Обработка дубликатов
Так как случайные числа могут повторяться, узел хранит поле `count` — сколько раз ключ был вставлен.  
Проверка `contains(x)` возвращает `True`, если узел найден и `count > 0`.

---

## Бенчмарк по заданию
В основной программе выполняется:

1) Создать **список** и **AVL‑дерево**  
2) Сгенерировать `N` случайных целых чисел в диапазоне `[-10^6; 10^6]` и добавить их в обе структуры  
3) Сгенерировать ещё `N` чисел и для каждого проверить наличие:
   - в списке (`x in list`)  
   - в AVL‑дереве (`tree.contains(x)`)
   Затем посчитать, сколько совпадений найдено (результаты обязаны совпасть).
4) Удалить проверку `x in list` и замерить время только для AVL‑проверки.

### Замеры времени
Используется `time.perf_counter()` — высокоточный счётчик для измерения коротких интервалов времени.

---

## Пример результатов замеров (один из запусков)
Пример (при `N = 30000`):

```
=== РЕЗУЛЬТАТЫ (list + AVL) ===
Вставка n=30000: 0.3532 сек
Проверка n=30000 (list+AVL): 4.8353 сек
Совпадений в list: 413
Совпадений в AVL : 413
Совпадают ли ответы: ДА

=== ТОЛЬКО AVL ===
Проверка n=30000 (только AVL): 0.0753 сек

=== СРАВНЕНИЕ ВРЕМЕНИ ПРОВЕРОК ===
list+AVL : 4.8353 сек
только AVL: 0.0753 сек
Ускорение (примерно): 64.19x
```

**Почему так быстрее:** проверка `x in list` для списка выполняется линейно по длине списка, а AVL‑проверка — логарифмически.

# Задача 2. Разбор и вычисление выражения

## Постановка
На вход поступает строка, содержащая:
- целые числа
- операции `+ - * /`
- круглые скобки `( )`

Выражение корректное. Нужно вывести значение выражения.  
Операция `/` выполняется как **целочисленное деление с усечением к нулю**.

### Почему нельзя просто `//`
В Python оператор `//` — это **floor division** (округление вниз), и для отрицательных чисел это отличается от “усечения к нулю”.  
Для требований лабораторной используется `int(a / b)` (или `math.trunc(a / b)`), чтобы получить усечение к нулю.

---

## Идея решения
Реализован классический подход:

1) **Токенизация**: превращаем строку в список токенов (числа, операторы, скобки)  
2) **Преобразование в RPN** (обратная польская нотация) алгоритмом *Shunting Yard*  
3) **Вычисление RPN** стеком

Дополнительно поддержан **унарный минус**:
- `-3` → число `-3`
- `-(2+3)` → преобразуется к `0 - (2+3)` внутри токенизации

---

## Примеры
```
1 + 2*3          = 7
(1 + 2) * 3      = 9
10/3             = 3
-(2+3)*4         = -20
7 + 8/(-3)       = 5   (целочисл. деление с усечением к 0)
```

# Задача 3*. Переменные

## Постановка
В выражении могут встречаться переменные — **одна маленькая латинская буква** (`a..z`).  
После ввода выражения программа:
1) находит все переменные
2) запрашивает значения у пользователя
3) вычисляет результат

Пример:
```
a*(b+3) - c/2
Введите a: 2
Введите b: 5
Введите c: 7
Результат: 2*(5+3) - 7/2 = 16 - 3 = 13
```

---

## Выводы
- AVL‑дерево обеспечивает быстрый поиск (логарифмический) по сравнению с линейным поиском в списке.
- Парсер выражений реализован через стандартную схему: токены → Shunting Yard → вычисление RPN стеком.
- Для деления выполнено требование “целочисленно, с усечением к нулю”.
